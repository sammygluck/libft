Multiple includes:
problems can occur in the following scenario:
test1.c has #include "test1.h"
test1.h has #include "test2.h"
test2.h has #include "test1.h"
Here an infinite loop will be created and will cause errors

How can we protect our files from such errors?

------
#ifndef __FT_TEST1_H__
# define __FT_TEST1_H__

#include "test2.h"

#endif
------

! This should become second nature when creating header files. Make sure you define the right macro names because if you don't and there are duplicates, it will omit the second one it encounters.

Structs:

example:
---------
struct	s_point
{
	int x;
	int y;
};

int	main(void)
{
	struct s_point a;
	struct s_point b;

	a.x = 10;
	a.y = 42;
	b = a;

	return (0);
}

--------


Now, we can simplify it even more. Instead of declaring 'struc s_point a;' each time we want to use one, we can make a typedef using this struct as follows:
-------
typedef struct s_point
{
	int x;
	int y;
}			t_point;

int	main(void)
{
	t_point a;
	t_point b;

	a.x = 10;
	a.y = 42;
	b = a;

	return (0);
}
------



Using pointers with structs:

-------
typedef struct s_point
{
	int x;
	int y;
}			t_point;

int	main(void)
{
	t_point a;
	t_point b;
	t_point *ptr;

	a.x = 10;
	a.y = 42;
	b = a;
	ptr = &a;
	(*ptr).x = 10; (one way to dereference. or:)
	ptr->x = 10;

	return (0);
}
